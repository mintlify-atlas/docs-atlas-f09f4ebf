description: Environment variables used in Hakim workspaces and templates
---

Hakim workspaces use environment variables for configuration, authentication, and runtime behavior. Variables are set through template parameters, agent configuration, or injected directly into containers.

## Coder agent variables

These variables are automatically set by the Coder agent and bootstrap scripts:

### CODER_AGENT_TOKEN

Authentication token for the Coder agent. Automatically injected by the template into the container environment.

**Set by:** Template (main.tf:1007)  
**Used in:** bootstrap.sh, container startup

### CODER_AGENT_URL

URL for the Coder agent to connect to the control plane. Automatically replaced with `host.docker.internal` when running on localhost to ensure connectivity from inside containers.

**Set by:** Template (main.tf:1008)  
**Used in:** bootstrap.sh, agent initialization  
**Default:** Workspace access URL

### CODER_AGENT_AUTH

Authentication method for the agent.

**Set by:** bootstrap.sh:80  
**Default:** `token`

### ARCH

System architecture of the provisioner.

**Set by:** Template (main.tf:1009)  
**Values:** `amd64`, `arm64`, etc.  
**Used in:** Binary downloads, image selection

**Set by:** opencode module (main.tf:170)

### ARG_OPENCODE_CONFIG

**Set by:** opencode module (main.tf:171)

### ARG_PRE_INSTALL_SCRIPT

Base64-encoded script to run before OpenCode installation.

**Set by:** opencode module (main.tf:172)

### ARG_POST_INSTALL_SCRIPT

Base64-encoded script to run after OpenCode installation.

**Set by:** opencode module (main.tf:173)

### ARG_AI_PROMPT

Base64-encoded initial task prompt for OpenCode.

**Set by:** opencode module (main.tf:180)

### ARG_SESSION_ID

Session ID to continue a previous OpenCode session.

**Set by:** opencode module (main.tf:181)  
**Optional:** Yes

### ARG_CONTINUE

Whether to continue the last OpenCode session.

**Set by:** opencode module (main.tf:183)  
**Default:** `false`

### ARG_PORT

Port for the OpenCode web server.

**Set by:** opencode module (main.tf:184)  
**Default:** `4096`

### ARG_HOSTNAME

Hostname for the OpenCode web server to bind to.

**Set by:** opencode module (main.tf:185)  
**Default:** `0.0.0.0`

## User-defined environment variables

These variables are injected from template parameters:

### Custom variables (user_env)

Any environment variables defined in the `user_env` JSON parameter are injected into the container. These are merged with default environment variables.

**Set by:** Template (main.tf:537, 1010)  
**Source:** `user_env` template parameter  
**Format:** JSON object

### Secret variables (secret_env)

Sensitive environment variables defined in the `secret_env` JSON parameter. These are handled identically to `user_env` but masked in the UI.

**Set by:** Template (main.tf:538, 1010)  
**Source:** `secret_env` template parameter  
**Format:** JSON object

## Variable precedence

When multiple sources define environment variables, they are merged in this order (later overwrites earlier):

1. Default environment variables (`MIX_HOME`, `HEX_HOME`, `MIX_ARCHIVES`)
2. Variables from `user_env` parameter
3. Variables from `secret_env` parameter
4. System variables (`CODER_AGENT_TOKEN`, `CODER_AGENT_URL`, `ARCH`)

This is implemented in main.tf:544:

```hcl
local.combined_env = merge(local.default_env, local.user_env, local.secret_env)
```

## Setting custom variables

To add custom environment variables to your workspace:

1. **Via user_env parameter** - For non-sensitive variables:
   ```json
   {
     "NODE_ENV": "development",
     "LOG_LEVEL": "debug"
   }
   ```

2. **Via secret_env parameter** - For sensitive data:
   ```json
   {
     "DATABASE_URL": "postgresql://user:pass@host:5432/db",
     "API_SECRET": "your-secret-here"
   }
   ```

3. **In shell profile** - Add to `~/.bashrc` or `~/.profile` for session-specific variables

4. **Via setup_script** - Export variables in the setup script for initialization-time configuration
