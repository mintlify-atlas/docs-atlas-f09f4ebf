title: .NET
description: Install .NET SDK versions using Mise
---

Installs .NET SDK using [Mise](https://mise.jdx.dev/) with support for multiple SDK versions and preview releases.

## Feature ID

```
dotnet
```

## Usage

Add the feature to your `devcontainer.json`:

```json
{
  "features": {
    "../../../features/src/dotnet": {
      "version": "9.0",
      "additionalVersions": "8.0,10.0-preview"
    }
  }
}
```

## Options

<ParamField path="version" type="string" default="9.0">
  Select the primary .NET SDK version.
  
  Supported values:
  - `latest` - Latest stable release
  - `9.0` - .NET 9.0
  - `8.0` - .NET 8.0
  - Any specific version number
</ParamField>

<ParamField path="additionalVersions" type="string" default="10.0-preview">
  Comma-separated additional .NET SDK versions.
  
  Install multiple SDK versions side-by-side. Supports preview versions by appending `-preview` to the version number (e.g., `10.0-preview`).
  
  Examples:
  - `"8.0,9.0"` - Install .NET 8.0 and 9.0
  - `"8.0,10.0-preview"` - Install .NET 8.0 and 10.0 preview
  - `"none"` - Don't install additional versions
</ParamField>

## What gets installed

This feature:

1. Installs required ICU libraries for .NET runtime
2. Installs the primary .NET SDK version via Mise
3. Installs any additional SDK versions specified
4. Normalizes preview version specifiers (removes `-preview` suffix for Mise)
5. Verifies all installed SDKs

## ICU library support

The feature automatically detects and installs the appropriate ICU (International Components for Unicode) library for your system. It checks for ICU versions in this order:

- libicu76, libicu75, libicu74, libicu73, libicu72, libicu71, libicu70
- Falls back to libicu-dev if no specific version is found

ICU is required for .NET's globalization and internationalization features.

## Environment variables

The feature sets up:

- `MISE_YES=1` - Auto-confirms Mise prompts
- `MISE_DATA_DIR=/usr/local/share/mise` - Mise data directory
- .NET binaries are available via Mise shims in `/usr/local/share/mise/shims`

## Multiple SDK versions

When multiple SDK versions are installed, you can:

1. Use `dotnet --list-sdks` to see all installed SDKs
2. Create a `global.json` file to pin a specific version for a project
3. All SDKs are accessible through the same `dotnet` command

## Preview versions

Preview versions are specified using the `-preview` suffix (e.g., `10.0-preview`). The feature automatically normalizes this to the format expected by Mise.

## Dependencies

This feature requires:

- `ghcr.io/devcontainers/features/common-utils`
- Mise feature (included in Hakim base image)

## Verification

After installation, verify .NET is working:

```bash
dotnet --version
dotnet --list-sdks
```
